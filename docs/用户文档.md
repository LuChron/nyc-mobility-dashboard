# NYC Urban Mobility Dashboard 用户文档

## 1. 系统简介

NYC Urban Mobility Dashboard 是一个面向纽约城市出行数据的交互式可视化大屏。系统基于 NYC TLC Trip Record Data，展示 Yellow Taxi、Green Taxi 和 HVFHV（Uber / Lyft 等高容量网约车）三类服务在时间、空间、OD 流向和出行行为上的差异。

用户可以通过左侧筛选器切换数据源、时间范围、区域和指标；中间地图、右侧 KPI 与排行、底部图表会根据当前筛选条件同步更新，帮助用户观察不同服务类型和不同区域的出行模式。

当前前端版本以可视化原型为主：地图边界已经使用真实 NYC Taxi Zone GeoJSON，页面中的统计数值仍主要由 mock 聚合数据驱动。后续接入真实 JSON 后，交互方式保持一致。

## 2. 页面整体结构

页面采用一屏式大屏布局，主要分为五个区域：

| 区域 | 位置 | 作用 |
|---|---|---|
| 顶部信息栏 | 页面顶部 | 展示项目名称、数据来源说明和总体概览指标 |
| 左侧筛选器 | 页面左侧 | 控制数据源、时间、区域和地图指标 |
| 主地图 | 页面中部 | 展示 NYC Taxi Zone 空间分布、热点区域和选中区域的 OD 流向 |
| 右侧指标与排行 | 页面右侧 | 展示核心 KPI、Top Pickup Zones 和 Top OD Routes |
| 底部图表区 | 页面底部 | 展示月度趋势、星期小时热力图、服务类型对比和费用/距离分布 |

## 3. 顶部信息栏

顶部信息栏用于说明当前大屏的主题和数据背景。

- `NYC Urban Mobility Dashboard`：项目标题。
- `Taxi & For-Hire Vehicle Trip Analytics`：说明本系统关注出租车和网约车出行分析。
- 顶部概览指标：展示数据规模、覆盖月份、服务类型等概览信息，用于帮助用户快速了解当前数据集范围。

顶部信息栏本身不提供点击交互，主要承担数据背景说明和页面导航识别作用。

## 4. 左侧筛选器

左侧筛选器是整个页面最主要的交互入口。用户调整筛选条件后，中间地图、右侧 KPI、排行列表和底部图表都会重新计算并刷新。

### 4.1 Data Source

数据源筛选用于切换当前分析的出行服务类型。

| 选项 | 含义 |
|---|---|
| `All` | 同时查看 Yellow Taxi、Green Taxi 和 HVFHV 的综合结果 |
| `Yellow` | 只查看黄色出租车数据 |
| `Green` | 只查看绿色出租车数据 |
| `HVFHV` | 只查看高容量网约车数据 |

使用场景：

- 对比出租车与网约车的空间热点差异。
- 观察不同服务类型的平均费用和平均距离差异。
- 查看某一类服务在机场、曼哈顿核心区或布鲁克林等区域的表现。

### 4.2 Time Selection

时间筛选用于控制当前分析的年份、月份范围和日期类型。

| 控件 | 选项 | 含义 |
|---|---|---|
| `Year` | `2025 - 2026`、`2025`、`2026` | 选择分析年份范围 |
| `Month` | `All Months (16)`、`Q1`、`Q2`、`H2` | 选择月份区间 |
| 日期类型 | `Weekday`、`Weekend` | 切换工作日或周末模式 |

切换时间条件后，系统会更新总行程量、峰值小时、月度趋势、星期小时热力图和排行结果。

### 4.3 Region Selection

区域筛选用于控制当前关注的纽约行政区和 Taxi Zone。

| 控件 | 含义 |
|---|---|
| `Borough` | 选择行政区，例如 Manhattan、Queens、Brooklyn，或查看全部区域 |
| `Taxi Zone` | 在当前 borough 范围内选择具体 Taxi Zone |

交互规则：

- 选择 borough 后，Taxi Zone 下拉列表会更新为该 borough 内的区域。
- 切换 borough 时，已选 Taxi Zone 会自动重置为 `All Zones`。
- 选择具体 Taxi Zone 后，地图会高亮该区域，OD 流向图会显示与该区域相关的主要路线。

### 4.4 Metric Selection

指标筛选用于控制地图颜色和 Top Pickup Zones 排行的统计口径。

| 指标 | 含义 | 适合分析的问题 |
|---|---|---|
| `Pickup` | 上车量 | 哪些区域是主要出发地 |
| `Dropoff` | 下车量 | 哪些区域是主要到达地 |
| `Trip Count` | 行程量 | 哪些区域整体出行需求更高 |
| `Average Fare` | 平均费用 | 哪些区域对应的出行费用更高 |
| `Average Distance` | 平均距离 | 哪些区域对应的行程距离更长 |

切换指标后，中间地图的颜色强度和右侧 Top Pickup Zones 的排序会随之变化。

### 4.5 Reset Filters

`Reset Filters` 用于恢复默认筛选条件：

```text
Data Source: All
Year: 2025 - 2026
Month: All Months
Day Type: Weekday
Borough: All Boroughs
Taxi Zone: All Zones
Metric: Pickup
```

当用户多次筛选后想回到初始全局视图，可以点击该按钮。

## 5. 主地图区域

主地图是大屏的核心视图，用于展示 NYC Taxi Zone 层面的空间分布和 OD 流向。

### 5.1 地图颜色

地图中每个 Taxi Zone 的颜色表示当前指标的强弱。颜色越接近高亮色，说明该区域在当前指标下数值越高；颜色越暗，说明数值越低。

颜色对应的指标由左侧 `Metric Selection` 控制。例如：

- 选择 `Pickup` 时，颜色表示上车量强度。
- 选择 `Dropoff` 时，颜色表示下车量强度。
- 选择 `Average Fare` 时，颜色表示平均费用高低。
- 选择 `Average Distance` 时，颜色表示平均距离长短。

### 5.2 热点点位

地图上的圆点表示当前筛选条件下排名靠前的热点区域。点位旁边会显示区域名称，便于用户快速识别主要出行热点，例如 Midtown Center、JFK Airport、LaGuardia Airport 等。

### 5.3 地图点击

用户可以点击地图上的 Taxi Zone：

- 如果当前没有选中区域，点击后会选中该 Taxi Zone。
- 如果再次点击同一个 Taxi Zone，会取消选择并回到全部区域视图。
- 选中区域后，地图会高亮该区域，并显示与该区域相关的 OD 流向线。
- 右侧 KPI、Top Pickup Zones 和 Top OD Routes 会根据选中区域同步变化。

### 5.4 OD 流向线

OD 流向线表示区域之间的主要出行连接。当前版本在选中具体 Taxi Zone 后显示与该区域相关的主要路线。

流向线用于回答：

- 当前区域的主要出行目的地有哪些？
- 机场与曼哈顿核心区之间是否存在高频连接？
- 某个区域是更偏出发地，还是更偏到达地？

### 5.5 地图工具按钮

地图右侧提供三个工具按钮：

| 按钮 | 功能 |
|---|---|
| `+` | 放大地图 |
| `-` | 缩小地图 |
| `◎` | 重置地图缩放，并取消当前选中的 Taxi Zone |

地图本身支持拖拽和平移缩放，便于查看局部区域。

### 5.6 Tooltip

鼠标悬停在地图区域上时，会显示该区域的 tooltip，内容包括：

- Taxi Zone 名称。
- Borough。
- 当前指标数值。

如果该区域在当前原型数据中没有统计值，tooltip 会提示当前原型未提供该区域 mock 数据。

## 6. 右侧指标与排行

右侧区域由 KPI 卡片和两个排行列表组成，用于快速读取当前筛选结果下的重点信息。

### 6.1 KPI 卡片

KPI 卡片展示当前筛选条件下的核心指标。

| 卡片 | 含义 |
|---|---|
| `Total Trips` | 当前条件下的总行程量 |
| `Avg Fare` | 当前条件下的平均费用 |
| `Avg Distance` | 当前条件下的平均行程距离 |
| `Peak Hour` | 当前条件下的高峰小时段 |

每张卡片还包含变化比例和迷你趋势线，用于辅助判断指标变化方向。

### 6.2 Top Pickup Zones

`Top Pickup Zones` 展示当前筛选条件和当前指标下排名靠前的 Taxi Zone。

列表信息包括：

- 排名序号。
- Taxi Zone 名称和 Location ID。
- 横向条形进度，用于表示相对强度。
- 当前指标值。

交互方式：

- 点击列表中的区域名称，会选中对应 Taxi Zone。
- 地图会同步高亮该区域。
- OD 路线和其他图表会随选中区域更新。

### 6.3 Top OD Routes

`Top OD Routes` 展示当前筛选条件下行程量最高的 OD 路线。

每条路线由起点和终点组成，例如：

```text
Midtown Center -> JFK Airport
```

该列表用于发现高频出行连接，尤其适合观察机场、曼哈顿核心区和主要商业区之间的流动关系。

## 7. 底部图表区

底部图表区从时间、服务类型和出行行为角度补充解释地图结果。

### 7.1 Monthly Trend - Trip Count

月度趋势图展示 2025-01 至 2026-04 各月份的行程量变化。

用途：

- 观察出行需求是否随月份变化。
- 比较不同数据源下的趋势差异。
- 判断某些月份是否存在明显峰值或低谷。

切换左侧数据源、年份、月份和区域后，趋势图会同步更新。

### 7.2 Weekday x Hour Heatmap - Trip Count

星期小时热力图展示一周中不同星期、不同小时段的出行强度。

用途：

- 识别早晚高峰。
- 对比工作日和周末的出行节奏。
- 观察夜间、午间或晚高峰时段的需求差异。

颜色越亮，表示该星期和小时组合下的行程量越高。

### 7.3 Source Comparison - Trip Count

服务类型对比图用于比较 Yellow Taxi、Green Taxi 和 HVFHV 三类服务。

当前对比指标包括：

- `Trip Count`：行程量。
- `Avg Fare`：平均费用。
- `Avg Distance`：平均距离。

该图适合回答：

- 哪类服务贡献了更多行程？
- 网约车与出租车的平均费用是否不同？
- 不同服务的平均行程距离是否存在差异？

### 7.4 Fare / Distance Distribution

费用/距离分布图展示不同服务类型在费用或距离区间上的分布情况。

图表右上角提供两个切换按钮：

| 按钮 | 含义 |
|---|---|
| `Fare (USD)` | 查看费用分布 |
| `Distance (mi)` | 查看行程距离分布 |

用途：

- 观察短途和长途行程占比。
- 比较不同服务类型在高费用区间或长距离区间的差异。
- 判断机场等长距离场景是否对费用/距离分布产生影响。

## 8. 典型使用流程

### 8.1 查看全市总体出行热点

1. 保持默认筛选条件。
2. 查看主地图颜色和热点点位。
3. 观察右侧 `Top Pickup Zones`，识别上车需求最高的区域。
4. 查看底部月度趋势和星期小时热力图，了解时间变化。

### 8.2 比较出租车和网约车差异

1. 在 `Data Source` 中选择 `Yellow`。
2. 观察地图、KPI 和底部图表。
3. 切换到 `HVFHV`。
4. 对比两种服务在热点区域、平均费用、平均距离和出行趋势上的差异。

### 8.3 分析机场相关出行

1. 在 `Borough` 中选择 `Queens`。
2. 在 `Taxi Zone` 中选择 `JFK Airport` 或 `LaGuardia Airport`。
3. 查看地图上的 OD 流向线。
4. 查看右侧 `Top OD Routes`，判断机场与哪些区域连接最强。
5. 切换 `Average Fare` 或 `Average Distance`，观察机场行程的费用和距离特征。

### 8.4 对比工作日和周末

1. 在 `Time Selection` 中选择 `Weekday`。
2. 记录 KPI、Peak Hour 和热力图分布。
3. 切换到 `Weekend`。
4. 对比总行程量、高峰小时和星期小时热力图的变化。

### 8.5 查找高费用或长距离区域

1. 在 `Metric Selection` 中选择 `Average Fare`。
2. 观察地图高亮区域和右侧排行。
3. 切换到 `Average Distance`。
4. 判断高费用区域是否也对应长距离行程。

## 9. 当前版本说明

当前前端已经完成以下交互：

- 数据源筛选。
- 年份、月份和工作日/周末筛选。
- Borough 和 Taxi Zone 筛选。
- 地图指标切换。
- 地图点击选中 Taxi Zone。
- Top Pickup Zones 点击选中 Taxi Zone。
- 地图放大、缩小和重置。
- 费用分布与距离分布切换。
- 多视图随筛选条件联动刷新。

当前版本仍待完善的部分：

- 图表数值仍主要来自 mock 聚合数据，真实 JSON 数据接入还在后续阶段。
- `View All Zones` 和 `View All Routes` 按钮目前主要作为界面占位，尚未打开完整列表弹窗。
- OD 流向线当前只在选中具体 Taxi Zone 后显示，避免默认状态下路线过多影响阅读。
- 当前 borough 下拉主要提供 Manhattan、Queens、Brooklyn，后续可以扩展 Bronx、Staten Island、EWR 等完整区域。

## 10. 运行方式

本地运行前端：

```bash
cd frontend
npm install
npm run dev
```

启动后，在浏览器打开终端中显示的本地地址即可访问大屏。

构建检查：

```bash
cd frontend
npm run build
```
